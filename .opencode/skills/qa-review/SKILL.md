Use this skill when the user wants a QA pass on a completed AutoSpec ticket or sprint.

- Read `skills/claude/qa-review.md` and treat it as the detailed workflow definition.
- Apply change-appropriate QA rather than a one-size-fits-all checklist.
- Keep `CLAUDE.md` and repo-specific rules active while reviewing.
- Update backlog state only after verification conclusions are clear.
